# 📱 اجرای لوکالِ «دستیارِ تصمیمِ معاملاتی» روی اندروید — بدونِ کلادفلر، بدونِ رمِ سنگین

> **پاسخِ کاملِ User Note:** سایتِ کلادفلری سنگین شده و رمِ ۲GB کفایت نمی‌کند. این پوشه
> راهِ اجرای همان ابزار را **به‌صورتِ کاملاً لوکال روی گوشیِ اندرویدِ شما** فراهم می‌کند —
> بدونِ Node، بدونِ Vite، بدونِ Wrangler، بدونِ Workerd. **مصرفِ رمِ اثبات‌شده: ~۲۴MB.**

---

## 🧠 چرا سایتِ فعلی سنگین بود و راهِ حل چیست؟ (تحلیلِ خطی + غیرخطی)

### ریشهٔ سنگینی (تفکرِ خطی)
اندازه‌گیریِ دقیق نشان داد که **خودِ منطقِ استراتژی‌ها سبک است** (کلِ `web_tool/src/*.ts` فقط
~۴۰۰KB و ۵۰۰۰ خط). رمِ ۲GB را این‌ها مصرف می‌کنند، نه استراتژی‌ها:

| جزء | مصرفِ رم |
|---|---|
| فرآیندِ `vite build` | ۵۰۰MB–۱.۵GB |
| `wrangler pages dev` (شبیه‌سازِ Workerd کلادفلر) | ۴۰۰–۸۰۰MB |
| `esbuild` (باندل‌سازی) | ۲۰۰–۴۰۰MB |
| `node_modules` | ۳۰۰MB+ دیسک |

⇒ **مسئله معماری است، نه محاسباتی.** کلِ این زنجیره فقط برای «شبیه‌سازیِ محیطِ کلادفلر»
لازم بود، نه برای اجرای منطقِ تصمیم.

### راهِ حل (تفکرِ غیرخطی)
پروژه **قبلاً** موتورِ تصمیم را به **JavaScriptِ خالص** پورت کرده بود (پوشهٔ `apk/`؛
فایلِ `engine.js` = پورتِ دقیقِ `live_engine.py` با **صفر اختلافِ عددی** روی ۱۵۰k کندل).
همچنین سایت **هیچ CSV سنگینی را در حافظه نمی‌خوانَد** — دادهٔ زنده مستقیم از Yahoo می‌آید.

پس برای اجرای لوکال، به هیچ‌کدام از آن زنجیرهٔ سنگین نیازی نیست. فقط لازم است:
1. **چند فایلِ استاتیک** (`apk/www/` که خودکفاست).
2. **یک وب‌سرورِ فوق‌سبک** که فقط فایل بدهد + پروکسیِ Yahoo باشد (تا CORS مانع نشود).
3. همان **موتورِ JS خالصِ** اثبات‌شده که در مرورگرِ گوشی اجرا می‌شود.

`server.py` این کار را با **فقط کتابخانهٔ استانداردِ پایتون** انجام می‌دهد (صفر `pip install`).

## ✅ راهِ ۱ (توصیهٔ اصلی): Termux + سرورِ سبکِ پایتون

**بهترین تعادل:** سبک، پایدار، دقیقاً «نمای سایت» در مرورگرِ گوشی.

### گام‌به‌گام
1. **نصبِ Termux** (از F-Droid یا GitHub — نه از Google Play که نسخه‌اش قدیمی است):
   - https://f-droid.org/packages/com.termux/
2. **گرفتنِ فایل‌های پروژه روی گوشی** (یکی از سه راه):
   - **الف) کلونِ گیت:**
     ```bash
     pkg update && pkg install -y git python
     git clone https://github.com/golshandvr-del/Scientific-Research.git
     cd Scientific-Research/local-mobile
     ```
   - **ب) اگر فقط همین پوشه را می‌خواهید** (سبک‌تر) — بعداً بخشِ «بستهٔ کمینه» را ببینید.
3. **اجرا با یک دستور:**
   ```bash
   bash start.sh
   ```
   این اسکریپت خودش python را (در صورتِ نبود) نصب می‌کند، سرور را بالا می‌آورد و
   مرورگرِ گوشی را روی آدرس باز می‌کند.
4. **در مرورگرِ گوشی برو به:**
   ```
   http://localhost:8080
   ```

همین. کارتِ XAUUSD و EURUSD با ماشینِ حالتِ ۴-وضعیتی (خنثی / نزدیک‌شدن / ورود / مدیریت)
روی دادهٔ زنده کار می‌کنند.

### اجرای دستی (بدونِ start.sh)
```bash
cd Scientific-Research/local-mobile
python server.py           # پیش‌فرض پورت 8080
# یا پورتِ دلخواه:
PORT=9000 python server.py
```

### نکاتِ Termux
- برای اینکه هنگامِ خاموشیِ صفحه سرور قطع نشود: `termux-wake-lock` را قبلِ اجرا بزنید.
- برای بازکردنِ خودکارِ مرورگر: بستهٔ `termux-api` را نصب کنید (`pkg install termux-api`).
- توقفِ سرور: در ترمینالِ Termux کلیدهای `Ctrl+C`.

## ✅ راهِ ۲: دسترسی از گوشی به سرورِ روی کامپیوترِ خانگی (شبکهٔ محلی)

اگر یک کامپیوتر/لپ‌تاپ در همان وای‌فای دارید و نمی‌خواهید روی گوشی چیزی نصب کنید:

1. روی کامپیوتر:
   ```bash
   cd local-mobile
   HOST=0.0.0.0 python server.py
   ```
2. آدرسِ IP کامپیوتر را در شبکه پیدا کنید (مثلاً `192.168.1.20`).
3. در مرورگرِ گوشی برو به:  `http://192.168.1.20:8080`

مزیت: گوشی هیچ باری نمی‌کشد. عیب: کامپیوتر باید روشن و در همان شبکه باشد.

«اپِ همیشگیِ روی صفحهٔ اصلی» بهتر است ولی ساختش یک‌بار زحمت دارد.

## 📦 بستهٔ کمینه (اگر نمی‌خواهید کلِ ریپو ~۶۰۰MB را کلون کنید)

روی گوشی فقط این‌ها لازم است (بدونِ پوشهٔ سنگینِ `data/` و `.git/`):
```
local-mobile/server.py      ← سرور
local-mobile/start.sh       ← راه‌انداز
apk/www/                    ← فایل‌های اپ (index.html, engine.js, app.js, strategies/, sample_data.json)
```
می‌توانید فقط این دو پوشه را با هر روشِ انتقالِ فایل (مثلاً `scp`، اشتراکِ فایل، یا
`git clone --depth 1`) به گوشی ببرید. `server.py` به‌طورِ خودکار `../apk/www` را پیدا و سرو می‌کند.

> **نکته:** اگر خواستید فایل‌های اپ را کنارِ خودِ `server.py` بگذارید، یک پوشهٔ
> `local-mobile/webroot/` بسازید و محتوای `apk/www/` را در آن کپی کنید؛ سرور به‌طورِ
> خودکار `webroot/` را در اولویت قرار می‌دهد.

## ❓ رفعِ اشکال
- **صفحه باز نمی‌شود:** مطمئن شوید سرور در حالِ اجراست و آدرس دقیقاً `http://localhost:8080` است.
- **دادهٔ زنده نمی‌آید:** ممکن است اینترنتِ گوشی به Yahoo دسترسی نداشته باشد؛ اپ خودکار به
  **دادهٔ نمونهٔ آفلاین** (`sample_data.json`) برمی‌گردد تا موتور همچنان نمایش داده شود.
- **پورت اشغال است:** با `PORT=9000 python server.py` پورتِ دیگری بدهید.
- **در Termux صفحه هنگامِ قفلِ گوشی قطع می‌شود:** `termux-wake-lock` را بزنید.
