# 📱 اجرای سایتِ کامل روی گوشی با Termux (راهنمای کامل و ساده)

این پوشه به شما اجازه می‌دهد **کلِ سایتِ دستیارِ تصمیمِ معاملاتی** را — با **همان منطقِ
کاملِ web_tool** (همهٔ لایه‌های رکورد، همهٔ کارت‌ها، `/api/decision`، مدیریتِ معامله و …) —
مستقیماً روی گوشیِ خودتان و بدونِ اینترنتِ بیرونی (به‌جز دریافتِ قیمتِ زنده) اجرا کنید.

> ✅ **این نسخهٔ ساده‌شده نیست.** دقیقاً همان اپِ Hono در `web_tool/` است که فقط طوری
> بسته‌بندی شده تا روی Node/Termux اجرا شود. هیچ لایه‌ای حذف یا ساده نشده است.

> ✅ **روی گوشی نیازی به build یا `npm install` نیست.** فایلِ آمادهٔ `app.bundle.mjs`
> که کلِ منطقِ سایت + خودِ Hono را در خود دارد، از قبل در گیت است. شما فقط کلون/آپدیت
> می‌کنید و اجرا می‌کنید.

---

## 🧩 در این پوشه چه چیزهایی هست؟

| فایل | کار |
|---|---|
| `server.mjs` | سرورِ سبک (فقط با ماژول‌های داخلیِ Node، بدونِ وابستگی). سایت را روی شبکهٔ محلی سرو می‌کند. |
| `app.bundle.mjs` | **باندلِ آمادهٔ کلِ منطقِ سایت** (همهٔ روترها/لایه‌ها + Hono). این را دستی ویرایش نکنید. |
| `cf-shim.mjs` | پلی که `serveStatic` کلادفلر را روی Node ممکن می‌کند (فقط هنگامِ ساختِ باندل لازم است). |
| `build.mjs` | ساختِ دوبارهٔ `app.bundle.mjs` (فقط روی کامپیوتر/سندباکس؛ روی گوشی لازم نیست). |
| `start.sh` | **راه‌اندازیِ سرور روی گوشی** (نصبِ Node + wake-lock + اجرا). |
| `update.sh` | **به‌روزرسانیِ دستی** بدونِ کلونِ دوباره. |
| `stop.sh` | توقفِ کاملِ سرور + آزادسازیِ wake-lock. |

## 🚀 مرحلهٔ ۱ — نصبِ ابزارها در Termux (فقط بارِ اول)

Termux را باز کنید و این‌ها را خط‌به‌خط بزنید (بعدِ هر خط Enter):

```bash
pkg update -y && pkg upgrade -y
pkg install -y git nodejs termux-api
```

- `git` : برای گرفتنِ پروژه.
- `nodejs` : موتورِ اجرای سرور.
- `termux-api` : برای wake-lock (تا با خاموش‌شدنِ صفحه، سرور قطع نشود).

> اگر هنگامِ نصب سؤالی پرسید، `y` بزنید و Enter کنید.

## 📥 مرحلهٔ ۲ — کلونِ پروژه (فقط بارِ اول)

```bash
cd ~
git clone https://github.com/golshandvr-del/Scientific-Research.git
```

حالا کلِ پروژه در پوشهٔ `~/Scientific-Research` است.

> اگر ریپو خصوصی بود و نامِ کاربری/رمز خواست، به‌جای رمز باید یک
> **GitHub Personal Access Token** بزنید (از GitHub → Settings → Developer settings →
> Personal access tokens بسازید).

## ▶️ مرحلهٔ ۳ — اجرای سایت

```bash
cd ~/Scientific-Research/local-mobile
bash start.sh
```

بعد از چند ثانیه چیزی شبیهِ این می‌بینید:

```
  روی همین گوشی      : http://localhost:8080
  در شبکهٔ محلی (LAN) — این آدرس را به دوستتان بدهید:
      http://192.168.1.42:8080
```

- **روی همین گوشی:** مرورگرِ گوشی را باز کنید و بروید به `http://localhost:8080`
- سرور تا وقتی `Ctrl+C` نزنید یا Termux را نبندید، **روشن می‌ماند**.

> `start.sh` به‌صورتِ خودکار wake-lock می‌گیرد؛ یعنی حتی با خاموش‌شدنِ صفحهٔ گوشی،
> سرور قطع نمی‌شود.

---

## 👥 مرحلهٔ ۴ — دیدنِ سایت توسطِ دوستتان (شبکهٔ محلی / LAN)

اگر شما و دوستتان به **یک مودم/وای‌فای مشترک** وصل باشید:

1. سرور را با `bash start.sh` اجرا کنید.
2. آدرسِ `http://192.168.x.x:8080` را که در خروجی چاپ می‌شود به دوستتان بدهید
   (شمارهٔ دقیق در گوشیِ شما ممکن است فرق کند — همان که زیرِ «در شبکهٔ محلی» چاپ شد).
3. دوستتان همان آدرس را در مرورگرِ گوشیِ خودش باز کند.

> **نکته:** سرور روی `0.0.0.0` گوش می‌دهد، پس روی کلِ شبکهٔ محلی در دسترس است.
> اگر دوستتان نتوانست وصل شود، معمولاً به‌خاطرِ **حالتِ AP/Client Isolation** مودم است
> (بعضی مودم‌ها ارتباطِ مستقیمِ دو گوشی را می‌بندند)؛ آن گزینه را در تنظیماتِ مودم خاموش کنید.

## 🌙 مرحلهٔ ۵ — اجرای دائم در پس‌زمینه

**راهِ ساده (توصیه‌شده):** کافی است `start.sh` را اجرا کنید و بگذارید Termux باز بماند.
با wake-lock، حتی اگر صفحه خاموش شود سرور زنده می‌ماند. برای اینکه اندروید Termux را
نکُشد، یک‌بار این را هم بزنید (اکتیو نگه‌داشتنِ سرویسِ Termux):

```bash
termux-wake-lock
```

و در تنظیماتِ اندروید، برای Termux **بهینه‌سازیِ باتری (Battery optimization)** را روی
«محدود نشود / Don't optimize» بگذارید تا سیستم آن را نبندد.

**راهِ کاملاً پس‌زمینه (اختیاری، پیشرفته‌تر):** اگر می‌خواهید حتی با بستنِ پنجرهٔ Termux
هم سرور بماند، از `nohup` استفاده کنید:

```bash
cd ~/Scientific-Research/local-mobile
termux-wake-lock
nohup bash start.sh > server.log 2>&1 &
```

- خروجی در فایلِ `server.log` ذخیره می‌شود (`cat server.log` برای دیدن).
- برای بستنِ کامل بعداً: `bash stop.sh`

## 🔄 مرحلهٔ ۶ — به‌روزرسانیِ دستی (دفعاتِ بعد)

هر وقت پروژه تغییر کرد و خواستید نسخهٔ جدید را بگیرید، **دیگر کلون نکنید**؛ فقط:

```bash
cd ~/Scientific-Research/local-mobile
bash update.sh      # فقط تغییراتِ جدید را می‌گیرد (git pull)
bash start.sh       # سرور را دوباره اجرا/ری‌استارت می‌کند
```

- `update.sh` هیچ‌چیز را خودکار به‌روز نمی‌کند؛ **کاملاً دستی** است، دقیقاً هر وقت خودتان بخواهید.
- چون باندلِ آمادهٔ سایت هم در گیت است، بعد از `update.sh` منطقِ جدیدِ سایت به‌طورِ خودکار می‌آید.

## ❓ رفعِ اشکالِ سریع

| مشکل | راه‌حل |
|---|---|
| `node: command not found` | `pkg install -y nodejs` |
| `git: command not found` | `pkg install -y git` |
| `EADDRINUSE ... 0.0.0.0:8080` (پورت اشغال است) | دیگر رخ نمی‌دهد: سرور خودکار پورتِ بعدی (8081، 8082، ...) را می‌گیرد و بالا می‌آید. فقط به خطِ «روی همین گوشی: http://localhost:PORT» نگاه کنید تا پورتِ واقعی را ببینید. اگر خواستید همه را ببندید: `bash stop.sh` |
| `fuser: Bad system call` | بی‌خطر است و دیگر رخ نمی‌دهد؛ اسکریپت‌ها به‌جای `fuser` از `pkill` استفاده می‌کنند. اگر هنوز دیدید، یک‌بار `bash update.sh` بزنید تا اسکریپت‌ها به‌روز شوند |
| دوستم وصل نمی‌شود | مطمئن شوید هر دو روی یک وای‌فای هستید + Client Isolation مودم خاموش باشد |
| قیمت‌ها نمی‌آید | اینترنتِ گوشی روشن باشد (قیمتِ زنده از Yahoo گرفته می‌شود) |
| با خاموشیِ صفحه قطع می‌شود | `pkg install termux-api` و اجرای `termux-wake-lock`؛ و خاموش‌کردنِ بهینه‌سازیِ باتری برای Termux |
| صفحه بعد از چند دقیقه هنگ می‌کرد/از دسترس خارج می‌شد | ✅ رفع شد (نشتیِ حافظهٔ فرانت‌اند). فقط `bash update.sh` بزنید؛ چون این رفع در فایل‌های `public/static` است، **نیازی به ساختِ دوبارهٔ باندل نیست** |

## 🔧 برای توسعه‌دهنده: ساختِ دوبارهٔ باندل (فقط روی کامپیوتر/سندباکس)

اگر کدِ `web_tool/src/` را تغییر دادید، باندلِ آماده را دوباره بسازید (نه روی گوشی):

```bash
cd web_tool && npm install          # فقط یک‌بار، برای داشتنِ esbuild و hono
cd ../local-mobile && node build.mjs # app.bundle.mjs را می‌سازد
git add app.bundle.mjs && git commit -m "rebuild bundle" && git push
```

سپس روی گوشی فقط `bash update.sh` می‌زنید و نسخهٔ جدید می‌آید.

## ⚡ سرعتِ بارگذاری روی گوشی (مهم)

قبلاً روی گوشی، صفحه گاهی تا **۲ دقیقه** خالی می‌ماند (پس‌زمینه می‌آمد ولی کارت‌ها
ظاهر نمی‌شدند). علت این بود که مرورگر تا کاملِ **همهٔ ۱۲ دارایی** هیچ کارتی نشان
نمی‌داد و هر دارایی مستقیم از Yahoo داده می‌گرفت (بدونِ کش، چون کشِ `cf:` فقط روی
Cloudflare کار می‌کند نه روی Node/Termux). پنج بهبودِ زیر این را حل کرد:

| # | بهبود | اثر |
|---|-------|-----|
| ۱ | **کشِ حافظه‌ایِ مستقل از محیط** (`cache.ts`: TTL + SWR + single-flight) | داده‌های تکراری فقط یک‌بار از شبکه گرفته می‌شوند |
| ۲ | **de-dup درخواست‌ها** | چند کارتِ طلا که یک تایم‌فریم می‌خواهند، **یک** fetchِ مشترک دارند |
| ۳ | **timeout روی هر fetch (۶ ثانیه)** | یک منبعِ کند دیگر کلِ صفحه را هنگ نمی‌کند |
| ۴ | **رندرِ تدریجی + کارت‌های مخفی بدونِ fetch** | کارت‌های سریع فوراً می‌آیند؛ هر کارتی که کاربر مخفی کند، یک دسته درخواستِ کمتر |
| ۵ | **پیش‌گرم‌سازی (pre-warm) در استارتِ سرور** | همان اولین کاربر هم کارت‌ها را تقریباً فوری می‌بیند |

**نتیجهٔ اندازه‌گیری‌شده روی همین باندل:**
- فهرستِ کارت‌ها (`/api/assets`) ≈ **۱۵ms** (فوری، بدونِ هیچ درخواستِ شبکه)
- هر کارت با کشِ گرم ≈ **۰.۱ تا ۰.۴ ثانیه** (قبلاً چند ثانیه، با ریسکِ تا ۲ دقیقه)

> نکته: منطقِ تصمیم‌گیریِ استراتژی‌ها **اصلاً تغییر نکرد**؛ فقط لایهٔ داده‌گیری
> سریع‌تر شد. خروجیِ هر کارت دقیقاً همان تصمیمِ قبلی است.

**اگر گوشی‌ات قوی است (رمِ زیاد):** می‌توانی همهٔ کارت‌ها را نمایان بگذاری؛ کش
فشارِ شبکه را مدیریت می‌کند. اگر شبکه‌ات ضعیف است، کارت‌های غیرِضروری را مخفی کن تا
حتی سریع‌تر شود.
