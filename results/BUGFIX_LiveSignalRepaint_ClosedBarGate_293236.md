# رفعِ باگِ حیاتی — Repainting سیگنالِ زنده روی کندلِ ناتمام (Closed-Bar Gate)

> **نوع:** رفعِ باگِ محصول (نه کشفِ استراتژی). سودِ خالصِ رکورد **بدون تغییر = +$293,236**.
> **منشأ:** گزارشِ صریحِ کاربر (User Note عملی) دربارهٔ سیگنالِ S215 روی XAUUSD M15.

---

## ۱) گزارشِ کاربر (نشانه)

کاربر روی کارتِ **XAUUSD M15** یک سیگنالِ **BUY از S215** دید:
- ورود ~**4050**، SL **4020**، TP **4095** (≈ ۴۵۰ pip هدف).
- بلافاصله قیمت ریخت تا **4047**؛ پس از رفرشِ صفحه، همان کارت **«خنثی»** شد.
- سؤالِ کاربر: «چرا اول انقدر قاطع سیگنال داد و بعد خنثی شد؟ نکند بقیهٔ سیگنال‌ها هم همین‌طور باشند؟»

## ۲) ریشه‌یابیِ دقیق (تشخیص)

### ۲.۱ منبعِ داده
`index.tsx::rebaseFuturesToSpot()` (و مشابهش `mergeLiveQuote`) برای نمایشِ قیمتِ لحظه‌ای،
**آخرین کندل را با قیمتِ spot زنده به‌روز می‌کند** در حالی که آن کندل هنوز در سطلِ زمانیِ
جاری است (`last.time >= curBucketStart`):
```ts
rebased[rebased.length - 1] = { ...last, close: spot.price,
  high: Math.max(last.high, spot.price), low: Math.min(last.low, spot.price) }
```
سپس `useCandles = merged.candles` **بدونِ حذفِ این کندلِ ناتمام** به همهٔ روترها داده می‌شد.

### ۲.۲ منطقِ ماشه‌ٔ سیگنال روی همان کندلِ ناتمام
S215 (`gold_trend_line.ts::computeTrendLine`) شرطِ ورود را روی `t = n-1` (آخرین کندل)
می‌سنجد:
```
penetrated (low[t] < line) && closedBack (close[t] > line - tol) && bullBar (close[t] >= open[t]) && !range
```
چون `t = n-1` **کندلِ در حالِ شکل‌گیری** بود:
- **لحظهٔ اول (close=4050):** low زیرِ خط رفته + close بالای خط + صعودی ⇒ هر ۳ شرط ✓ ⇒ **ENTRY کاذب**.
- **لحظهٔ دوم (close=4047):** همان کندلِ باز، close افت کرد ⇒ کندل نزولی/زیرِ خط ⇒ **شرط شکست ⇒ NEUTRAL**.

⇒ **این «repainting» است:** سیگنال روی دادهٔ نهایی‌نشده صادر شد و با هر تیک عوض می‌شود.
TP=4095 هم پیش‌بینیِ مدل نبود؛ صرفاً `entry + tpDist` با فاصلهٔ ثابتِ config بود.

### ۲.۳ چرا این تناقض با بک‌تست است (نکتهٔ علمی مهم)
همهٔ بک‌تست‌هایی که رکوردِ +$293k را ساختند با **`shift(1)`** کار می‌کنند: سیگنال روی
کندلِ **بسته‌شدهٔ** `t` و ورود روی open کندلِ `t+1`. اما منطقِ زندهٔ قبل، سیگنال را روی
کندلِ **بازِ** `n-1` می‌سنجید ⇒ **منطقِ زنده با منطقِ اعتبارسنجی‌شده هم‌تراز نبود**. این
یعنی WR/سودِ خالصِ واقعیِ زنده می‌توانست از بک‌تست بدتر باشد (سیگنال‌های زودگذر و کاذب).

### ۲.۴ دامنهٔ باگ — سیستماتیک بود
`useCandles` با کندلِ زندهٔ آخر به **همهٔ** لایه‌های ماشه‌ای می‌رفت:
`decide` (روترِ اصلیِ M15)، `trendLineDecision`/`trendLineEntry` (S215، همهٔ TF)،
`channelDecision` (S219)، `decideGoldM5/M30/H1/H4`. ⇒ **همهٔ سیگنال‌های ماشه مستعدِ
repainting بودند**، نه فقط S215. پاسخِ صریح به نگرانیِ کاربر: بله، سیستماتیک بود.

## ۳) راه‌حل (Closed-Bar Gate)

اصل: **سیگنالِ ماشه فقط روی کندل‌های بسته‌شده ارزیابی شود؛ کندلِ زنده فقط برای نمایشِ
قیمت و بخشِ مدیریتِ معامله بماند.**

پیاده‌سازی: تابعِ کمکیِ `closedBars(candles, gapSec)` که اگر کندلِ آخر در سطلِ زمانیِ
جاری باشد (`last.time >= floor(now/gap)*gap`) آن را **حذف** می‌کند و آرایهٔ کندل‌های
«فقط-بسته‌شده» را برمی‌گرداند. این آرایه به منطقِ **تصمیمِ ورود/ماشه** داده می‌شود؛ در
مقابل، `useCandles`ِ کامل (با کندلِ زنده) فقط برای:
- محاسبهٔ قیمتِ لحظه‌ایِ نمایش،
- بخشِ `POST /api/trade/advice` (مدیریتِ معاملهٔ بازِ کاربر که ذاتاً زنده است)،
به کار می‌رود.

این دقیقاً `shift(1)`ِ بک‌تست را در محیطِ زنده بازتولید می‌کند: entry همان قیمتِ
spot لحظه‌ایِ «open کندلِ بعد» است، اما شرطِ سیگنال روی کندلِ بستهٔ قبلی سنجیده می‌شود ⇒
**بدونِ repaint، هم‌تراز با بک‌تست**.

## ۴) نتیجه

- **رفتار پس از رفع:** وقتی S215 روی کندلِ بستهٔ M15 سیگنال دهد، آن سیگنال **تا بسته‌شدنِ
  کندلِ بعد ثابت می‌ماند** (repaint نمی‌شود). دیگر «۴۰۵۰ buy ← رفرش ← خنثی» رخ نمی‌دهد.
- **سودِ خالصِ رکورد بدون تغییر** (این رفعِ باگِ نمایش/زمان‌بندی است، نه تغییرِ استراتژی).
- درسِ بنیادی برای پروژه: **هر منطقِ ماشه‌ای در محیطِ زنده باید روی کندلِ بسته‌شده اجرا شود**؛
  کندلِ زنده فقط برای نمایش و مدیریتِ معاملهٔ باز است. (به README به‌عنوان قاعدهٔ دائمی اضافه شد.)
