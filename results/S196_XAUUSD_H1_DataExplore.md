# S196 — تحلیلِ اکتشافیِ XAUUSD H1 «بگذار داده حرف بزند» (پاسخِ User Note: تایم‌فریمِ هدف = H1)

> قانونِ #۱: هدف فقط سودِ خالصِ بیشتر (XAUUSD + EURUSD). WR≥40٪ فقط کفِ پذیرش.
> این نشست پاسخ به User Note است: «استراتژی‌ها را فقط روی M15 تست کرده‌ایم؛ شاید روی
> H1 که داده‌اش موجود است بازدهی داشته باشند. اول داده را بررسی کن و بگذار حرف بزند.»

## چیستی
- **نوع:** تحلیلِ اکتشافیِ خام (بدونِ معامله). ورودی: `data/XAUUSD_H1.csv`.
- **کد:** `strategies/s196_h1_data_explore.py`.

## یافته‌های داده (خام)

### [۱] پوششِ داده
- **۹۰٬۹۵۰ کندلِ H1** از **۲۰۱۱-۰۱-۰۳** تا **۲۰۲۶-۰۷-۱۶** (~۱۵.۵ سال).
- **نکتهٔ مهم:** دادهٔ H1 **بسیار عمیق‌تر** از M15 است (M15 فقط از ۲۰۲۰). این یعنی
  walk-forward روی H1 می‌تواند رژیم‌های بیشتری (۲۰۱۱–۲۰۲۶) را بپوشاند ⇒ آزمونِ
  ضدِ overfit سخت‌گیرانه‌تر و معتبرتر.

### [۲] مقایسهٔ ATR (بازتنظیمِ TP/SL و max_hold)
| تایم‌فریم | میانهٔ range | میانهٔ ATR14 |
|---|---:|---:|
| **H1** | ۳۳.۲ pip | ۳۸.۰ |
| **M15** | ۲۴.۱ pip | ۲۵.۹ |
| نسبت H1/M15 | **۱.۳۸×** | ۱.۴۷× |

⇒ **درسِ کلیدی (تأییدِ User Note):** هر کندلِ H1 دامنهٔ بزرگ‌تری دارد. TP/SLهای M15
روی H1 باید کمی بزرگ‌تر شوند و **`max_hold` بر حسبِ *تعداد کندل* باید تقسیم بر ۴ شود**
(چون هر کندلِ H1 = ۴ کندلِ M15). مثلاً `mh48` در M15 (=۱۲ ساعت) روی H1 می‌شود `mh12`.

### [۳] ⭐ Drift ساعتی — قوی‌ترین سیگنالِ H1
| ساعتِ UTC | میانگین (pip) | t-stat |
|---:|---:|---:|
| **1** | **+5.42** | **+4.7** ⭐ |
| **3** | +2.54 | +3.0 ⭐ |
| **0** | +3.85 | +2.7 ⭐ |
| 18 | +1.27 | +1.4 |
| 22 | +0.98 | +1.3 |

**تفسیر:** خوشهٔ **۰–۳ UTC (سشنِ آسیایی / پایانِ سشنِ نیویورک)** قوی‌ترین drift مثبتِ
پروژه روی H1 است — ساعتِ **۱ UTC با t=+4.7** استثنایی است. این همان مکانیزمِ
**S139 Overnight Drift** است اما با **پنجرهٔ ساعتیِ متفاوت**: در M15 ساعاتِ ۲۱–۲۳ UTC
غالب بود؛ روی H1 ساعتِ **۲۲/۲۳ ضعیف شده** و مرکزِ ثقل به **۰–۳ UTC** جابه‌جا شده.
⇒ **تأییدِ مستقیمِ User Note:** «هر تایم‌فریم بهبود/تنظیمِ متناسبِ خود را می‌خواهد.»

### [۴] Drift روزِ هفته
همه مثبت اما هیچ‌کدام قوی نیست (بیشترین: جمعه t=+1.6). ⇒ لایهٔ Monday (S140) روی H1
احتمالاً ضعیف است؛ باید تست شود ولی انتظارِ زیادی نداریم.

### [۵] Drift روزِ تقویمیِ ماه
Turn-of-Month (t=+0.9)، Mid-Month (t=+1.0)، End-of-Month (t=+0.6). همه مثبت اما ضعیف
روی رزولوشنِ H1 (چون کندل‌های کمتری در هر پنجره).

### [۶] بایاسِ کلی و ساختار
- میانگینِ بازدهِ هر کندلِ H1 = **+۰.۲۸۷ pip** (بایاسِ صعودیِ ساختاریِ طلا — تأییدِ درسِ
  بنیادیِ پروژه: «طلا بایاسِ صعودی دارد»).
- اتوکورلیشنِ lag-1 = **−۰.۰۱۵** (تصادفی/خنثی) ⇒ روی H1 استراتژیِ **momentum خالص**
  (روند-ادامه بدونِ ماشهٔ زمانی/ساختاری) بعید است لبه بدهد؛ **زمان-محورها و
  price-actionِ ساختاری** شانسِ بیشتری دارند.
