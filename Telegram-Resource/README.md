## اولویتِ بررسی (از باارزش‌ترین به کم‌ارزش‌ترین برای تحقیق)

1. **PDFها (کتاب/مقاله)** — ایدهٔ استراتژیِ خام؛ قابلِ ترجمه به قانونِ بک‌تست‌پذیر.
2. **فایل‌های `.mq4`/`.mq5` (سورسِ باز)** — منطقِ دقیقِ اندیکاتور/EA قابلِ استخراج و پیاده‌سازیِ مجدد در پایتون.
3. **فایل‌های `.txt` توضیحِ سیستم** — قوانینِ ورود/خروج به‌زبانِ ساده.
4. **`.set` (تنظیماتِ بهینه)** — پارامترهای پیشنهادیِ سازنده.
5. **`.ex4`/`.ex5` (کامپایل‌شده)** — فقط رفتارِ ظاهری؛ ارزشِ کمتر (سورس بسته).

### پروتکلِ بررسیِ هر فایل (تولیدِ `<filename>.md`)
برای هر موردِ بررسی‌شده، یک فایلِ MD **در همین پوشه** ساخته می‌شود که شامل:
1. **چیستی:** نوع (اندیکاتور/EA/کتاب)، تایم‌فریم/دارایی هدفِ سازنده.
2. **منطقِ استخراج‌شده:** قانونِ ورود، خروج، فیلترها، اندیکاتورهای پایه.
3. **ایدهٔ قابلِ آزمون برای پروژهٔ ما:** چگونه به قانونِ بک‌تست‌پذیر روی XAUUSD/EURUSD ترجمه می‌شود.
4. **ارتباط با پرتفویِ فعلی:** آیا لبهٔ نو می‌دهد یا صرفاً فیلترِ تأییدِ لایه‌های موجود است.
5. **نتیجهٔ بک‌تست (اگر آزموده شد):** Δ سودِ خالص، WR هر لایه (کفِ ۴۰٪)، تصمیمِ پذیرش/رد.

> **یادآوریِ صداقتِ علمی:** بیشترِ این سیستم‌های تجاری «no-repaint / 100% win» تبلیغ
> می‌شوند اما معمولاً یا repaint دارند یا روی هزینهٔ واقعی زیان‌ده‌اند. هر ادعا **فقط**
> پس از بک‌تستِ بدونِ look-ahead روی دادهٔ خودمان پذیرفته می‌شود.
