## ۲. منطقِ استخراج‌شده

هستهٔ ارزشمند = **دو الگوریتمِ مدیریتِ خروج** که به‌طور تمیز قابلِ استخراج‌اند:

1. **Trailing Stop (فاصلهٔ ثابت):**
   ```
   Buy:  newSL = Bid − TrailingStop·point ;  اگر newSL > OrderSL آنگاه به‌روزرسانی
   Sell: newSL = Ask + TrailingStop·point ;  اگر newSL < OrderSL آنگاه به‌روزرسانی
   ```
   (SL فقط در جهتِ سود حرکت می‌کند، هرگز عقب نمی‌رود.)

2. **Break-Even (انتقالِ SL به نقطهٔ سربه‌سر پس از سود):**
   ```
   Buy: اگر (Bid − BreakEvenAfter·point) ≥ OrderOpenPrice
        ⇒ SL = OpenPrice + BreakEvenPips·point   // قفلِ سودِ کوچک
   ```
3. **Auto-Lot ساده:** `Lot = (Balance / LotSize) × RiskFactor` (بدونِ لحاظِ فاصلهٔ SL —
   ساده‌تر و کم‌دقت‌تر از منطقِ ریسک-محورِ ما).

## ۳. ایدهٔ قابلِ آزمون برای پروژهٔ ما

- **Break-Even پس از X پیپ سود** یک قاعدهٔ مدیریتِ خروجِ بسیار متداول است که مستقیماً به
  بخشِ **«مدیریتِ معامله»** سایتِ ما (`trade_manager.ts`) مربوط می‌شود: پس از رسیدنِ قیمت
  به `BreakEvenAfter` پیپ سود، توصیهٔ «SL را به نقطهٔ ورود + چند پیپ ببر».
- **Trailing با فاصلهٔ ثابت** واریانتِ ساده‌ای است که می‌توان با trailing مبتنی بر ATR
  (موردِ `24786`) و منطقِ فعلیِ `tpsl_plan.py` مقایسه کرد: کدام سودِ خالصِ بیشتر می‌دهد.
