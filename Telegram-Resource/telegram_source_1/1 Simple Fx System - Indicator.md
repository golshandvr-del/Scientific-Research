## ۲. منطقِ استخراج‌شده (از سورسِ `.mq4`)

اندیکاتور دو خط رسم می‌کند (Resistance قرمز، Support آبی) و **کاملاً بر پایهٔ
`iFractals` استانداردِ MT4** است:

```
val1 = iFractals(MODE_UPPER, i)   // فرکتالِ سقف (Bill Williams)
اگر val1>0:  Resistance[i] = High[i]      // سطحِ جدید
وگرنه:       Resistance[i] = Resistance[i+1]  // نگه‌داشتنِ آخرین سطح

val2 = iFractals(MODE_LOWER, i)   // فرکتالِ کف
اگر val2>0:  Support[i] = Low[i]
وگرنه:       Support[i] = Support[i+1]
```
- یعنی: هر فرکتالِ سقفِ Bill Williams (سقفِ بالاترین در پنجرهٔ ۵ کندلی) → آخرین سطحِ
  مقاومت؛ متقارن برای حمایت. بینِ فرکتال‌ها، سطح ثابت نگه داشته می‌شود (خطِ پله‌ای).
- **iFractals تأخیرِ ۲ کندلی دارد** (برای تأییدِ فرکتال باید ۲ کندلِ بعد بسته شود) →
  ذاتاً non-repaint اگر با احتیاطِ ۲-کندلی استفاده شود.

## ۳. ایدهٔ قابلِ آزمون برای پروژهٔ ما

- **سطوحِ SR فرکتالی به‌عنوان TP/SL یا محلِ ورود:** آخرین Support/Resistanceِ فرکتالی را
  به‌عنوان هدفِ سود یا حدِ ضرر برای لایه‌های موجود به کار ببریم.
- **استراتژیِ شکست (breakout):** ورودِ Long وقتی قیمت آخرین مقاومتِ فرکتالی را با کندلِ
  بسته می‌شکند؛ متقارن Short. کاملاً بک‌تست‌پذیرِ non-repaint (با تأخیرِ ۲ کندلیِ فرکتال).
- **استراتژیِ بازگشت (bounce):** ورود از نزدیکیِ سطحِ حمایت/مقاومت با تأییدِ کندلِ برگشتی.
- ترجمهٔ `iFractals` به پایتون: سقفِ محلی که `High[i] > High[i±1], High[i±2]` (پنجرهٔ ۵ کندلی).

## ۴. ارتباط با پرتفویِ فعلی

- **لبهٔ نو:** ⚠️ محدود — SR فرکتالی مفهومِ پایه‌ای است و احتمالاً در پرتفوی به‌شکلی
  (swing high/low) وجود دارد.
- **فیلترِ تأیید / منبعِ سطح (بالقوه مفید):** سطوحِ فرکتالی می‌توانند **منبعِ SL/TP
  ساختاری** برای لایه‌های موجود باشند (به‌جای SL/TP ثابتِ pip). این می‌تواند WR یا R:R را
  بهبود دهد. **در نشستِ آینده به‌عنوان بهبودِ TP/SL ساختاری ارزشِ آزمون دارد.**
