# بررسیِ منبع: `1-2-3-pattern-day-trader/` (اندیکاتورِ الگوی 1-2-3)

> قانونِ شمارهٔ ۱: تابعِ هدف = **سودِ خالصِ بیشتر** (XAUUSD + EURUSD)؛ WR کفِ ۴۰٪ دارد.
> مشخصاتِ حساب: `CONTRACT_SIZE=100`، طلا `3.3pip`، کمیسیون=صفر.

دهمین موردِ بررسی‌شده (ترتیبِ الفباییِ `LC_ALL=C`).

---

## ۱. چیستی

| ویژگی | مقدار |
|---|---|
| **نوع** | پوشهٔ سیستم؛ هستهٔ آن اندیکاتورِ **`123PatternsV6/V7`** با **سورسِ باز (`.mq4`)** + مجموعه‌ای از اندیکاتورهای کمکیِ «Day_*» (Heiken Ashi, open line, signal arrow, …). |
| **الگوریتمِ اصلی** | تشخیصِ الگوی کلاسیکِ **1-2-3 (Reversal/Continuation)** بر پایهٔ ZigZag. |
| **تایم‌فریمِ هدفِ سازنده** | Setupِ پیش‌فرض: «15pip RangeBars Basic Setup» (day trading). |
| **پارامترها (از سورس)** | `ZigZagDepth=1`, `RetraceDepthMin=0.4`, `RetraceDepthMax=1.0`, `Target1Multiply=1.5`, `Target2Multiply=3.0`. |

## ۲. منطقِ استخراج‌شده (از سورسِ `123PatternsV6.mq4`)

الگوی 1-2-3 با سه نقطهٔ ZigZag تعریف می‌شود (نقاط `one`, `two`, `three`):
```
retracedepth = (three - two) / (one - two)          // نسبتِ اصلاح
شرطِ الگو:  RetraceDepthMin(0.4) < retracedepth < RetraceDepthMax(1.0)
```
- **ساختارِ صعودی (Bullish 1-2-3):** نقطه‌۱ = کفِ اولیه، نقطه‌۲ = سقفِ واکنشی، نقطه‌۳ =
  اصلاحِ نزولی که بینِ ۴۰٪ تا ۱۰۰٪ حرکتِ ۱→۲ برمی‌گردد (اما کفِ ۱ را نمی‌شکند).
- **سیگنالِ ورود = شکستِ نقطهٔ ۲** (BULLISH BREAK ABOVE #2): وقتی قیمت سقفِ نقطه‌۲ را می‌شکند،
  الگو تأیید و ورودِ Long صادر می‌شود؛ متقارن برای Short.
- **اهداف:** `Target1 = 1.5×` و `Target2 = 3.0×` اندازهٔ حرکتِ پایه (فاصلهٔ ۱→۲).
- ZigZag با پارامترهای استانداردِ `(depth, 5, 3)` محاسبه می‌شود.
- **⚠️ ZigZag = repaint.** نقطهٔ ۳ تا تأییدِ کاملِ swing جابه‌جا می‌شود؛ در بک‌تست باید فقط
  swingِ **بسته‌شده/تأییدشده** به کار رود تا look-ahead حذف شود.

## ۳. ایدهٔ قابلِ آزمون برای پروژهٔ ما

- **الگوی 1-2-3 breakout روی XAUUSD_M15 / EURUSD_M15:** با بازسازیِ non-repaint —
  از swing highs/lows تأییدشده (به‌جای ZigZagِ repaint) الگو را می‌سازیم؛ شرطِ
  `0.4 < retrace < 1.0`؛ ورود روی شکستِ نقطهٔ ۲؛ SL زیر نقطهٔ ۳؛ TP روی 1.5R / 3R.
- پارامترهای بهینه‌سازی: عمقِ swing، بازهٔ retrace (شاید 0.382–0.786 فیبوناچی)، مالتی‌پلایرِ هدف.
- این یک **الگوی ساختاری بک‌تست‌پذیر** است و دقیقاً به قانونِ عددی ترجمه می‌شود.

## ۴. ارتباط با پرتفویِ فعلی

- **لبهٔ نو (بالقوه):** الگوی 1-2-3 یک منطقِ ورودِ ساختاری (swing-based breakout) است که
  اگر پروژه هنوز آن را با این شرطِ دقیقِ retrace نیازموده باشد، ارزشِ آزمون دارد.
- **فیلترِ تأیید (بالقوه):** شرطِ «عمقِ اصلاحِ ۰.۴–۱.۰» می‌تواند فیلترِ کیفیتِ pullback برای
  ورودهای موجود باشد (رد کردنِ اصلاح‌های خیلی کم‌عمق یا بیش‌ازحد). **در نشستِ آینده باید
  همپوشانی با لایه‌های swing/breakoutِ موجود سنجیده شود.**
