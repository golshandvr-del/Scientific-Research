3. **ورود** — جهت (Long/Short) + **TP** + **SL** + R:R + احتمال؛ دکمهٔ «معامله را ثبت کردم».
4. **مدیریتِ معامله** — **فقط پس از ثبتِ معاملهٔ کاربر** فعال می‌شود؛ سود/زیانِ R، پیشرفت

## 🧠 درس‌های بنیادیِ پروژه (خلاصه)

- **کشف ≠ گرفتن:** سیگنال روند را می‌بیند ولی exit زودهنگام سود را پس می‌دهد؛ ضعفِ
  اصلی، خودِ **زمان‌بندیِ خروج** است (s116/s118).
- **طلا داراییِ با بایاسِ صعودیِ ساختاری است:** جریانِ SHORTِ سیستماتیک روی طلا شکننده
  است؛ تنها لبهٔ SHORTِ اثبات‌شده، SHORT-MA-Confluence با خروجِ سریع است.
- **«زمان» یک محورِ چند-بعدی است:** ساعتِ روز، روزِ هفته و روزِ تقویمیِ ماه هر کدام یک
  جریانِ سودِ ناهمبستهٔ مستقل می‌سازند (S139..S144).
- **WR تابع هدف نیست، اما کف دارد:** سودِ خالص = (نرخِ برد×میانگینِ برد) − (نرخِ باخت×میانگینِ باخت).
  فقط لایه‌های WR≥40 وارد رقابت می‌شوند و میانِ آن‌ها سودِ خالص بیشینه می‌شود.
